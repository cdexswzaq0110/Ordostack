# Codex Command: DEBUG MODE

用途：用最小修改修復可重現問題。

---

## Trigger Prompt

```text
請依 codex-workflows/commands/debug.md 修復以下問題：

[錯誤訊息]
[重現步驟]
[預期行為]
[實際行為]
```

---

## Objective

找出根因、修復問題、避免同類錯誤再次發生。

---

## Prerequisites

- 有錯誤訊息或重現步驟。
- 若無法重現，先回報缺少什麼。
- 不要盲目大量改檔。

---

## Process

1. 建立最小重現路徑。
2. 提出最多 3 個假設。
3. 用讀檔、測試或命令驗證假設。
4. 修改最小範圍。
5. 補回歸測試，或說明無法補測原因。
6. 更新 `progress.md`，記錄根因與修法。

---

## Output Format

```markdown
## 問題摘要

## 重現方式

## 根因

## 修改內容

## 測試結果

## 防止再發

## 本版不涵蓋
```

---

## Stop Rule

若連續 2 次嘗試仍無法解決，不要繼續亂改。回報：

- 已嘗試什麼。
- 目前證據。
- 可能原因。
- 下一步選項。
